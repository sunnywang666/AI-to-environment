# 自查与优化升级方案 v1

> 对照参考：MIT Technology Review《We did the math on AI's energy footprint》（技术标杆）、
> PBS News 两篇数据中心报道（严肃排版标杆）。

---

## 一、别人怎么做的（关键发现）

### MIT Technology Review（交互天花板）
- **滚动引擎**：GSAP + **ScrollTrigger**，`scrub:true`（滚动 1:1 驱动动画）+ `pin:true`（图固定、内容在"背后"演）。这是 scrollytelling 的核心。
- **视觉冲击**：**Three.js** 做 50×50 芯片网格 3D 动画（从中心 stagger 扩散）。
- **数据图表**：**Flourish** 平台 iframe 嵌入（地理分布图等）。
- **数字动效**：`proxy={progress:0}` + `gsap.to(...onUpdate)` 自定义计数（带千位分隔），倒计时同理。
- **打字机文字**：逐字 `substring` + 闪烁光标，模拟"AI 在回答"。
- **工程**：`--cst-vh` 变量 + JS 修正移动端 100vh bug；`text-shadow:2px 2px 2px #000` 让文字压在图片上仍可读；深蓝黑底 `#061620` + 黄/绿点缀；正文衬线、数据用 mono。

### PBS（严肃报道排版）
- **图片即叙事**：`srcset` 多分辨率 + **带署名图说**（谁拍的/何时/何地/机构），建立可信度。
- **性能**：`preconnect` 预连 CDN、`font-display:swap` 防字体抖动、`lazysizes` 懒加载。
- **静态也能高级**：无花哨动效，靠排版/图片/留白取胜；sticky 分享栏 + 反馈入口。

## 二、自查：我们现状 vs 参考

| 维度 | 参考做法 | 我们现状 | 差距 |
|---|---|---|---|
| 滚动引擎 | GSAP ScrollTrigger，pin+scrub，图随滚动逐帧演 | 自研 rAF 渐隐淡入淡出 | **缺核心 scrollytelling**（没有 pin 的 sticky 场景） |
| 视觉素材 | 大量实拍照片 / 卫星图 / 3D | **纯文字 + 矢量，0 张图** | **最大短板**（你已两次指出"没素材""很素"） |
| 数字动效 | proxy + onUpdate 计数 | monument 大数字是静态的，`data-count` 没接动画 | 易补 |
| 数据图表 | Flourish 嵌入 / 自研 D3 | 全是占位框 | 待做（任务5-8） |
| 明星交互 | 计算器/滑块，精致 | 口径滑块已达标 | ✅ 基准已立 |
| 移动端 | --vh 修正、srcset、lazyload | 无 | 待补 |
| 字体/性能 | preconnect、font-display swap、代码分割 | 仅有 display=swap | 小补 |
| 排版基调 | 深色底、mono 数据字、text-shadow 压图 | 已有深色 + mono | ✅ 接近 |

**一句话结论**：基调和单点交互（滑块）已达专业水准；真正拉开差距的是两件事——
①**没有"图随滚动演"的 pin 场景**（现在只是文字淡入），②**完全没有图像素材**。

## 三、优化升级方案（按优先级）

### P0 · 补两个最大短板

**P0-1 引入 GSAP + ScrollTrigger，做真正的 pin 场景**
- CDN 引入 GSAP（现已完全免费，含 ScrollTrigger）。
- 改造重点段为"图钉住 + 滚动驱动"：
  - 第2段：数字从"一次查询"→"一天"→"一年"→"一个国家"逐级放大（scrub 驱动）。
  - 第3段：地图 pin 住，滚动时依次点亮中美集群、淡入缺水图层。
  - 第6段：双轴折线随滚动逐段画出。
- 保留现有逐块渐隐做文字层，GSAP 只接管图形场景（两者不冲突）。

**P0-2 建立图像素材体系（解决"没素材"）**
- 新建 `assets/img/`，统一**带署名图说组件**（仿 PBS：来源/时间/地点/机构）。
- 需求清单：数据中心航拍、变电站、西弗居民/账单、宁夏戈壁+数据中心、卫星 before/after。
- Hero 升级：第1段用一张震撼大图或轻量 canvas/WebGL 网格（芯片/服务器阵列），文字用 text-shadow 压在其上。
- **来源策略待定**（见下方提问）：公共版权（NASA/USGS 卫星=公有领域、Unsplash/CC）+ 参考文章配图（注明出处，课程作业范畴）。

### P1 · 工程与细节（快速见效）
- 数字 count-up：用 `proxy + onUpdate`（或 GSAP）接 ScrollTrigger，进入视口才滚动累加。
- 移动端 `--vh` 修正 + 媒体查询降级（双联图→堆叠、横滑→点击）。
- `preconnect` 字体域、图片 `srcset` + `loading="lazy"`。
- `text-shadow` / 半透明遮罩，保证图上文字可读。

### P2 · 各段图表（沿用滑块质感，任务4-8）
- 第1段开场估算器、第2段排行 Hero、第3段双联地图、第4段多因素拆解+人物卡、第6段折线收尾。
- 图表统一：mono 数据字、三维度配色、实测实线/预测虚线、可点开来源。
- 复杂地理图可评估 Flourish 嵌入（快、专业）vs 自研 D3（风格统一）——倾向自研保持深色沉浸一致性。
